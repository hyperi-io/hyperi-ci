# How to add an environment

Adding an environment means deploying an existing topology to a new cluster.

## 1. Register the cluster with ArgoCD

If the cluster is not yet registered:

```bash
argocd cluster add <kubectl-context-name> --name <cluster-alias>
```

Verify: `argocd cluster list`

## 2. Add the environment to the ApplicationSet

Open `argocd/applicationsets/<topology-name>.yaml` and add the new cluster to the
generator list:

```yaml
spec:
  generators:
    - list:
        elements:
          - cluster: staging
            url: https://staging.k8s.example.com
          - cluster: production
            url: https://production.k8s.example.com
          # Add here:
          - cluster: dr-production
            url: https://dr.k8s.example.com
```

## 3. Add environment-specific value overrides (optional)

If the new environment needs different values, add a file under `values/`:

```bash
# values/<cluster-alias>.yaml
dfe-loader:
  replicaCount: 1   # DR runs reduced capacity
  resources:
    requests:
      cpu: 250m
```

Update the ApplicationSet template to mount environment values:

```yaml
  template:
    spec:
      source:
        helm:
          valueFiles:
            - values.yaml
            - ../../values/{{cluster}}.yaml
```

## 4. Validate and PR

```bash
git checkout -b add-env-dr-production
git add argocd/applicationsets/<topology-name>.yaml values/dr-production.yaml
git commit -m "feat: add dr-production environment to <topology-name>"
git push origin add-env-dr-production
```

## 5. After merge

ArgoCD detects the new Application generated by the ApplicationSet and syncs it
automatically (if auto-sync is enabled). Otherwise, trigger a manual sync:

```bash
argocd app sync <topology-name>-dr-production
```

## OpenTofu

If the new environment requires new cloud infrastructure (VPC, EKS cluster, IAM),
add it under `terraform/` and attach the `tofu plan` output to the PR.
